--- 0) TEST VERİSİ TEMİZLİĞİ (encoding) ---
Bozukluk: "Çay 1kg" DB'de "�ay 1kg" olarak duruyordu.
Ham baytlar: efbfbd 61 79 20 31 6b 67
  efbfbd = U+FFFD REPLACEMENT CHARACTER — yani 'Ç' (C3 87) mangle olmamış,
  KAYBOLMUŞ. Bu geri döndürülemez: veri kaynağında zaten yok edilmiş.

KÖK NEDEN: Gün 18'de test verisi PowerShell üzerinden gönderilmiş. PowerShell
5.1'de Set-Content/curl gövdesi sistem ANSI kod sayfasına düşer; 'Ç' bu kod
sayfasında temsil edilemeyince U+FFFD ile değiştirilir. Kayıp gönderim
anında olur — DB veya API kusuru değil.

TEKRARLAMAMAK İÇİN: JSON gövdesi Node ile UTF-8 dosyaya yazılıp curl'e
--data-binary "@dosya" ile verildi; gövde hiçbir noktada shell encoding'ine
maruz kalmadı. Doğrulama: dosyanın ilk baytları ... 22c387617920316b67 22 ...
(c387 = doğru UTF-8 'Ç').
